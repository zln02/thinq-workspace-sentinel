> 데이터 출처: UIS `urban_immune` DB (KCDC 인플루엔자 확진 + UIS 조기경보 risk_scores + 다층 선행신호)

## 1. 사전 포착(Early Warning) — 실데이터 검증 결과 ✅ 입증됨

UIS 조기경보 점수(`risk_scores.composite_score`/`alert_level`)가 KCDC 확진 피크보다
**얼마나 먼저 경보를 발령했는가**를 2025–26 인플루엔자 시즌 실데이터로 측정.

| 지역 | 조기경보 발령(ORANGE/RED) | 확진 피크 | **선행(리드타임)** |
|---|---|---|---|
| 부산 | 2025-10-26 | 2025-12-08 | **43일** |
| 세종·울산 | 2025-11-02 | 2025-12-08 | **36일** |
| 경북·대전·전북·충남·경남·인천·제주 | 2025-11-09 | 2025-12-08 | **29일** |
| 광주·전남·충북·대구 | 2025-11-16 | 2025-12-08 | **22일** |
| 서울·경기 | 2025-11-23 | 2025-12-08 | **15일** |
| 강원 | 2025-12-14 | 2025-12-08 | -6일 (놓침) |

- **17개 시도 중 16곳에서 확진 피크 15~43일 전에 선제 경보** → 사전 포착 실증.
- **선행지표 융합**: 발령 시점에 하수 인플루엔자 RNA(KOWAS)=100, 감기약 구매(네이버 쇼핑)=67,
  검색량(데이터랩)=43 이 먼저 상승 → 확진 통계보다 3~4주 앞섬. 단일 지표가 아닌 다층 신호.
- **알람 정확도**: 상승 국면 알람 TP 100%(알람 2주 후 확진 2배↑, 예: 충남 137→258).
  하강 국면은 후행 오경보(FP) 발생 → `alert_level` 자동 해제로 일부 완화.
- **계절성 현실성**: 월별 per_100k 6월 13.9 → 12월 396.9 → 4월 23.3 = 실제 겨울 인플루엔자 패턴 재현.

### End-to-End 동작 검증 (실서버 :8003)
```
부산 선택(조기경보 RED) + 정상 센서값(가스90·23℃·50%)
  → tier ALERT (선제 상향) → governance auto → 코웨이 TURBO 자동 가동(code 200)
지역 미선택 + 동일 센서값 → MONITOR, 제어 없음
```
→ **센서가 멀쩡해도 외부 확산이 임박하면 미리 공기청정기를 켠다** = 사전 예방 작동 확인.

### 개선 적용(코드)
- 기존 `external_live.py`는 `MAX(per_100k)`(과거 최악치 **정적 룩업**)를 boost에 사용 — "사전 포착" 아님.
- → `risk_scores`(실시간 조기경보 점수) 기반으로 교체. replay(시즌 재현)/live(현재 실시간) 이중 모드.
- 응답에 리드타임·선행신호·확진피크일 노출 → 대시보드에서 "왜 미리 켜는지" 설명 가능.

## 2. 대상감 평가 — 강점 (심사기준 직격)

| 심사 포인트 | 본 시스템 근거 |
|---|---|
| **실제 작동(실물 시연)** | 아두이노 센서 → 라파이 브릿지 → GCP 백엔드 → **실제 코웨이 공기청정기 제어** 전 구간 실연동 |
| **데이터 신뢰성** | KCDC 실확진 + UIS 조기경보 + 4종 선행지표(하수·OTC·검색·기온) 융합, 시뮬 아님 |
| **사전 예방 가치** | 15~43일 선행 포착 → 요양병원 집단감염 **발생 전** 환기·정화 선제 대응 |
| **과학적 근거** | Rudnick-Milton 재호흡률 기반 Wells-Riley 감염확률 + 법정 IAQ 1000ppm 기준 |
| **실무 의사결정 반영** | 하이브리드 거버넌스(ALERT/HIGH_RISK 자동, CRITICAL 관리자 승인) |
| **완성도(UX)** | 실시간 대시보드·음성 안내·수동 제어·경보 해제 시 자동 복귀 |
| **B2G 적합성** | 9개 법령 매핑(요양병원), ISMS-P 지향, 조달 납품 품질 목표(p95<500ms) |

## 3. 실무 적용성 — 한계 & 보완 과제 (정직 평가)

| # | 한계 | 영향 | 실무 보완 방향 |
|---|---|---|---|
| 1 | **CO2 센서 미탑재** (현 MQ2 가스/온습도 대체) | Rudnick-Milton 핵심 입력 부재 → 데모는 가스/온습도 트리거 | MH-Z19B 도착 시 교체(코드 준비됨). "실측 CO2" 표현은 교체 후에만 |
| 2 | **조기경보가 외부 UIS 의존** | 본 제품 단독으론 risk_score 생성 불가 | UIS 연동 계약 or 자체 선행신호 수집 파이프라인. 현재는 KCDC/KOWAS 공개데이터 기반 |
| 3 | **하강 국면 후행 오경보(FP)** | 피크 후에도 경보 유지 | risk_score 모델 자체 개선 + 하강 감지 로직. 현재 alert_level 자동해제로 일부 완화 |
| 4 | **감염자수(I) 미측정** → PoI에 가정값(I=1) | 절대 감염확률 정확도 제한 | 출입/재실 카운트 센서 연동 시 실측화 |
| 5 | **코웨이 IoCare 클라우드 지연(수초)** | 실시간성 한계 | 로컬 ThinQ 직제어(LG 가전) 또는 Matter 로컬 제어가 이상적 |
| 6 | **1개 병동 실센서, 나머지 임의데이터** | 스케일 미검증 | 다병동 센서 증설 + 부하 테스트(p95<500ms 목표 검증) |
| 7 | **보안(ISMS-P)** | 코웨이 계정 .env 관리, 감사로그·토큰 회전 미흡 | 시크릿 매니저, 접근 감사로그, 개인정보 영향평가 |
| 8 | **강원 사례(-6일) 놓침** | 1개 시도 사전 포착 실패 | 지역별 임계 보정. 정직하게 "16/17 성공"으로 프레이밍 |

## 4. 결론

- **사전 포착은 실데이터로 입증됨** — 16/17 시도 15~43일 선행, end-to-end 자동 가동 확인.
- **대상감 충분** — "실물 작동 + 실데이터 + 과학적 근거 + 사전 예방"의 4박자. 특히 실물 코웨이 제어가 차별점.
- **실무 수준은 PoC 단계** — CO2 센서 교체·다병동 확장·ISMS-P 보안·UIS 연동 정식화가 상용화 전제.
- **시연 정직 가드**: replay 모드는 "겨울 시즌 재현"으로 명시, MQ2=상대가스 트리거(CO2 아님), 코웨이=실기기.
