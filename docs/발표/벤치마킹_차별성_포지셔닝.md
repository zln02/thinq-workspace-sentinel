# 벤치마킹 & 차별성 포지셔닝 (발표·QA 직결)

> 작성: 2026-06-09 / 근거: 2024~2026 실제 제품 웹 조사(LG·코웨이·삼성·다이슨·Awair·Honeywell·DCV·Matter)
> 핵심: **"공기질 자동제어"는 차별점이 아니다. "외부 역학융합 선제대응 + 감염관리 자동증빙"이 차별점이다.**

---

## 1. 경쟁 비교표 — 무엇이 이미 있고, 무엇이 비었나

| 기능 | 이미 상용화 (차별점 ❌) | 우리만 (차별점 ✅) |
|---|---|---|
| PM2.5 감지 → 공청기 자동 풍량 | 코웨이·삼성·샤오미·**LG 퓨리케어** | — |
| CO2 → 환기 자동(DCV) | 빌딩공조 30년 표준(ASHRAE 62.1)·다이슨 | — |
| 멀티가전 루틴("미세먼지↑→공청기 ON") | **LG ThinQ ON**·SmartThings·Matter | — |

| **Wells-Riley 감염위험도 → 8종 가전 차등 처방** | 학술논문만 | ✅ 빈영역 |

**한 줄 결론:** LG ThinQ ON·퓨리케어도 **"공기 나빠지면 켜기"= 사후 반응형**. 우리의 **"유행 감지 시 공기 깨끗해도 미리 켜기"= 선제 예측형**은 LG·경쟁사 모두 안 한다. (LG 퓨리케어엔 CO2 센서조차 없음)

## 2. "그럼 왜 아무도 안 했나?" — 빈영역의 이유 (= 우리 해자)

아무도 안 한 건 **가치가 없어서가 아니라, 가전사가 구조적으로 못 해서**다:
1. **효과 인과 입증 어려움** — "선제 가동→감염 감소"를 RCT로 증명한 연구 거의 없음. 가전사는 못 파는 효능은 마케팅 안 함.
2. **"감염 예방" = 의료기기/효능표시 규제** — 가전사는 본능적으로 회피.
3. **B2C ROI 안 맞음** — 선제 가동은 전기·필터 비용↑, 효과 불확실 → 매스마켓 무가치.
4. **크로스도메인** — 가전(LG)+역학(질병청)+감염관리(의료)를 한 회사가 못 묶음.

→ 이 4개 장벽은 **가전사에겐 벽, B2B SW 솔루션에겐 해자.** LG가 직접 하기 애매한 영역을 SW 레이어가 메우는 전형적 기회. (협업 파트너십 모델도 가능: LG 가전 인프라 + 우리 감염 인텔리전스)

---

## 3. ⚠️ 함정 회피 — 포지셔닝 전환 (가장 중요)

**"감염을 예방한다"로 팔면 죽는다** (증명 불가 + 의료기기 규제 트리거). 대신:

| ❌ 함정 (단정) | ✅ 사업 (증명 가능) |
|---|---|
| "감염을 예방/진단한다" | "감염관리를 **자동화·문서화**한다" |
| "AI가 막아준다" | 선제 **환기·공기질 관리** + 자동 기록 |
| 효능 주장 | **적정성평가 감염관리 점수 자동 충족 → 수가 직결** |

요양병원은 **적정성평가 점수 = 수가(돈)**라, "감염관리 자동 증빙"은 증명도 쉽고 즉시 지불가치. **"선제 감염 대응"은 후킹 비전, 실제 지불근거는 컴플라이언스 자동화 + 행정부담 제거 + 보호자 안심.**

## 4. 예상 반론 → 답변 (QA 카드)

**Q. "이거 LG ThinQ에 이미 있는 기능 아니에요?"**
> "ThinQ ON이 미세먼지 감지해 공청기 켜는 건 맞습니다. 그건 '나빠지면 대응'하는 사후형입니다. 저희는 두 단계 앞섭니다 — 질병청 지역 유행지수가 오르면 공기가 아직 깨끗해도 선제로 환기를 강화하고, 그 대응 이력을 적정성평가 증빙으로 자동 문서화합니다. 외부 역학데이터를 가전 제어 루프에 넣은 상용 제품은 현재 없습니다."

**Q. "선제 가동이 진짜 감염을 줄이나요? 증명됐어요?"**
> "감염 '예방' 효과의 인과 증명은 파일럿 단계 과제로 정직하게 둡니다. 저희가 지금 파는 가치는 그게 아니라 '감염관리 활동의 자동화·증빙'입니다 — 적정성평가 점수, 행정부담 절감, 환기 이력 자동 기록처럼 즉시 측정 가능한 가치입니다."

**Q. "그냥 비싼 공기청정기 아닌가요?"**
> "공기청정기는 현재 공기만 봅니다. 저희는 외부 역학 + 실내 센서 + 재실 정보를 결합해 '이 공간이 지금 감염 전파에 얼마나 취약한가'를 계산하고, 거기에 맞춰 여러 벤더 가전을 동시 처방합니다. 단일 기기가 아니라 공간 단위 감염 리스크 오케스트레이션입니다."
