# 방어논리 — "그냥 가전이 매일 이상적 환경 만들면 되잖아?" 에 대한 반박 방어

> 심사/질의 대비 핵심 논리. PPT엔 이상(full) 비전, 시연엔 실제 가능분만.

---

## 0. 한 줄 요지
**좋은 공기질은 ① 공짜가 아니고(비용) ② 하나가 아니다(병원체별 상충).**
그래서 "항상 최대"는 비싸고, 때론 더 위험하고, 무엇보다 가습/제습처럼 **상충**해서 불가능하다.
우리는 *외부 위협이 올 때 · 그 병원체에 맞는 환경을 · 필요한 만큼만 · 근거를 남기며* 만든다.

---

## 1. 반박: "24/7 가전 최대로 돌려서 항상 이상적 환경 유지하면 외부 데이터·AI 필요 없잖아?"

### 🔑 결정타 — 가습과 제습은 정반대다
- **인플루엔자**: 저온·저습에서 비말 안정성↑ → **가습 50%RH** 가 방어 (Lowen 2007)
- **노로바이러스**: 표면·저습 환경 → **제습 45%** + 살균이 방어
- 가습기와 제습기를 **동시에 최대로 돌릴 수 없다.** 어느 쪽으로 환경을 만들지 정하려면
  **"지금 무엇이 유행하는지"(외부 감염병 데이터)를 반드시 알아야 한다.**
- → 외부 데이터는 옵션이 아니라 **알고리즘의 필수 전제.** "항상 최대"는 물리적으로 성립 불가.

### 보강 4축
| "그냥 24/7 최대" 의 문제 | 우리 답 |
|---|---|
| **① 비용/에너지** | 전 병동 공청 TURBO·환기 MAX 상시 = 전기료·필터비 폭발. 요양병원 운영비 민감(B2G) → 위험할 때만 = 절감(병원장 ROI 직결) |
| **② 의학적 역효과** | 환기 무조건 MAX = 외부 미세먼지·꽃가루·한파/폭염 유입 → 노인 저체온·열사병·호흡기 자극. "좋은 공기질"은 외부 날씨·병원체와 **상충** |
| **③ 쾌적/수용성** | 24/7 터보 = 소음·강풍·과건조로 노인 수면·생활 방해. 비현실적 |
| **④ 선제성** | "매일 최대"조차 *언제 위협이 오는지* 모름. 외부 조기경보 = 확진피크 **최대 43일 전** 포착 → 그때만 강화. 평상시 절약 + 위협시 선제 |
| **⑤ 증빙(B2G)** | "왜 이때 이 세팅인가" 근거(외부+센서+모델) 로깅 → 적정성평가·수가 증빙. 무지성 최대는 근거 없어 증빙 불가 |

## 2. 알고리즘 구조 (3단)

```
[입력] 외부 조기경보(질병청 확진 + 하수 KOWAS + 검색 데이터랩 + 약국 OTC + 기온 KMA)
        → "어느 병원체가 유행 임박인가"
       + 실내 센서(CO2 · PM2.5 · 온습도 · 재실) + 카메라(인원수)
        ↓
[판단] 병원체 식별 → Wells-Riley/Rudnick-Milton 감염확률(PoI) → 5-Tier
        tier = max(센서 tier, 외부 boost tier)     ← 센서 정상이어도 외부 위협이면 선제 상향
        ↓
[출력] 병원체 × tier → 가전 차등 제어 (Smart Protocol)
        위험도 '비례' — 항상 최대 아님:
        MONITOR=대기 / CAUTION=공청 약(LOW) / ALERT↑=공청 TURBO+송풍 / CRITICAL=승인 후 최대
```

### 외부 경보등급 → 가전 매핑 (대시보드 '상관 패널'과 일치)
| 외부 UIS 경보 | 실내 tier | 실제 가전 작동 |
|---|---|---|
| 🟢 GREEN 정상 | MONITOR | 대기 (에너지 절약) |
| 🟡🟠 YELLOW/ORANGE 주의·경계 | CAUTION | **공기청정 LOW** 자동 |
| 🔴 RED 심각 | ALERT | **공청 TURBO + 에어컨 송풍 환기** |
| (내부 악화) CRITICAL | CRITICAL | 전 병동 터보 · **관리자 승인** |

## 3. 필수 가전 (감염예방 기여도순)

1. 🥇 **환기(Ventilation)** — 에어로졸·CO2 희석(Wells-Riley의 Q). 공기감염 예방 **1순위**. ACH↑.
2. 🥈 **공기청정기** — PM2.5·에어로졸 HEPA 여과. *(시연 실제 제어 ✅ 코웨이)*
3. 🥉 **온습도(가습/제습/냉난방)** — 병원체 생존율·점막 방어 조절(인플루 50%RH 등). 폭염 노인 사망 예방.
4. 보조 — 로봇청소기 살균(노로/CDI 표면), 스타일러(옴/린넨 살균).

> Smart Protocol은 8종 병원체(COVID-19·인플루엔자·RSV·노로·결핵·폐렴구균·CDI·옴)별로
> 목표 온/습도·환기율·공청 강도를 차등 적용한다.

## 4. PPT(이상) vs 시연(실제 가능) 분리

| | 📊 PPT 발표자료 (full 비전) | 🖥️ 실제 시연 (doable) |
|---|---|---|
| **가전** | 8종 병원체별 차등 Smart Protocol | **코웨이 공기청정기 실제 ON/TURBO/LOW** + 나머진 "제어계획" 화면(시뮬 라벨) |
| **에어컨** | 송풍 환기 보조(SmartThings) | 토큰 연결 시. *현재 미연결 → 시뮬 표시* |
| **데이터** | 질병청+하수+검색+약국+기온 5층 + 학술근거 | UIS 외부 조기경보 라이브 + 선제 시나리오 토글 |
| **센서** | CO2·PM·온습도·재실 풀스택 | RPi 201호(온습도·재실) + 카메라 인원 + 코웨이 실측 PM |
| **스토리** | "공기질 완화 → 감염예방"의 과학적 체계 + ROI/증빙 | 외부 RED → tier 점프 → **코웨이 실제 가동** 시연 |

### 학술 근거 (PPT용)
- **Wells-Riley / Rudnick-Milton**: 재호흡 CO2 분율 기반 감염확률 모델 (환기↑ → 감염확률↓).
- **Lowen et al. 2007**: 인플루엔자 전파의 온·습도 의존성(저온·저습 시 전파↑).
- **ASHRAE / REHVA**: 의료시설 환기율(ACH) 권고 — 공기감염 격리 ACH 6~12.

## 5. 시연 시나리오 (3분)
1. **평상시** — 외부 GREEN, 전 병동 정상, 가전 대기(에너지 절약 강조).
2. **선제 발령** — 「선제 시나리오 발령」 토글 → 외부 부산 RED(확진피크 43일 전) → 전 병동 선제 ALERT.
3. **자동 대응** — 코웨이 공기청정기 **실제로** TURBO 가동(센서는 아직 정상인데도 = 사전예방).
4. **상관 패널** — "외부 경보 → 가전 차등제어"에서 현재 등급(심각) 하이라이트 → "항상 최대가 아니라 위험도 비례" 설명.
5. **병원장 뷰** — 사전예방 성과(선행 43일 · 선제대응 N건 · 위험 억제) + ROI.
6. **해제** — 토글 OFF → 평상시 복귀.
