# 【BX】

## Page 1. 추진배경 — 우리가 선택한 LG전자의 핵심 난제

### LG전자의 난제 (한 문장)
> **LG ThinQ 가전은 '제어'는 잘하지만, "지금 왜 켜야 하는가"를 판단하는 공간 위험 인텔리전스가 없다.** 그래서 가전은 늘 "나빠진 뒤" 반응하는 사후 대응에 머문다.

### Why now (왜 지금인가)
- **초고령화 + 요양병원 집단감염 급증**: 전국 요양병원 1,342개소(HIRA 2024), 면역취약 고령 입소자 밀집 → 호흡기 감염 1건이 집단감염·휴원·사망으로 직결.
- **실내공기질관리법 강화 + 적정성평가 감염관리 영역 신설** → 요양병원이 감염관리를 "증빙"해야 수가를 받는 시대.
- **공공 역학데이터 개방 + ThinQ API 전면 개방(2024.12)** → 외부 감염 데이터를 가전 제어에 연결할 기술 토대가 이제야 마련됨.

### Why us (왜 우리 팀인가)
- 외부 감염병 조기경보 시스템(UIS) 연계 자산 보유 → 질병청 역학 신호를 실시간으로 당겨 쓸 수 있는 유일한 팀.
- 실센서(아두이노)+실가전(코웨이·삼성) end-to-end 실증 완료.

### BX 경쟁사 분석 (배점 직결 — 벤치마킹 실데이터)
| 경쟁/대안 | 무엇을 하나 | 한계 | Sentinel 차별 |
|---|---|---|---|
| LG ThinQ ON·퓨리케어 AI | PM·VOC 감지 → 공청기 자동 가동, 멀티가전 루틴 | **사후 반응형**("나빠지면 켜기"), CO2 센서 없음, 역학 미연동 | 외부 유행 감지 시 **선제** 가동 |
| 코웨이·삼성·다이슨 | 공기질 자동 풍량(일부 CO2) | 단일 기기·사후·감염위험 모델 없음 | 공간 단위 감염위험 오케스트레이션 |
| 빌딩 BMS·DCV | CO2 기반 환기 자동(ASHRAE 표준) | 환기만, 감염위험 점수화·역학 없음 | Wells-Riley 위험등급 기반 차등 처방 |
| Awair·Honeywell 헬시빌딩 | 공기질 모니터링·경보 | **측정·알림만**, 자동제어·역학 없음 | 경보→자동 제어→자동 증빙 |
| KT/SKT 스마트병원 | 병원 IoT 플랫폼 | 감염 특화·역학융합 없음 | 요양병원 감염관리 특화 |

> **핵심:** "외부 역학데이터를 가전 제어에 연결한 상용 제품은 현재 0건"(웹 조사 6회 전무). 빈영역의 원인은 가전사의 구조적 제약(효능규제·크로스도메인)이며, 이는 B2B SW 솔루션에겐 진입장벽(해자)이 된다.

## Page 2. 고객감동목표 — 핵심 난제와 관련된 집중 과제

### 집중 과제 (한 문장)
> **지역 감염이 병동에 닿기 전, 공간을 자동으로 준비시키고, 그 모든 대응을 적정성평가 증빙으로 자동 문서화한다.**

### 감동 포인트 (Before → After)
| 구분 | Before (현재 요양병원) | After (Sentinel) |
|---|---|---|
| 위험 인지 | 증상 발생 후 사후 인지 | 외부 유행 15~43일 선행 포착 |
| 대응 판단 | ICN의 경험·직감 | 위험등급(5-Tier) 자동 산정 |
| 가전 운영 | 수동 지시, 따로 작동 | 위험도 기반 8종 자동 처방 |
| 기록·보고 | 수기 일지·행정부담 | 대응 이력 자동 로그·CSV 증빙 |
| 보호자 신뢰 | 구두 설명 | 실시간 안심 PWA |

> 감동의 본질 = **"막아준다"가 아니라 "알아서 미리 관리하고, 증명까지 해준다"** — ICN의 행정부담을 없애고, 원장의 적정성평가 점수를 지킨다.

## Page 3. 데이터 수집전략 (이미 강함 — 요약)
- **외부 역학(선행신호)**: 질병청 KCDC 확산 위험, 하수 감시(KOWAS), OTC 의약품(네이버), 검색 데이터랩, 기온(KMA) → UIS 조기경보 `risk_scores`(GREEN~RED).
- **실내 환경**: 아두이노(온습도·가스) + 코웨이 실측(PM·CO2) → `sensor_readings`.
- **전처리**: 이동평균 평활, 결측 폴백, 외부신호 추세기반 FP 차단(`_trend_adjust`).
- **신뢰성 근거**: 2025-26 인플루엔자 시즌 17개 시도 중 16곳 15~43일 선행 포착 백테스트.

## Page 4. 타겟 고객 — 페르소나 · 대표욕구 · Actor 정의서

### 페르소나 ① 감염관리간호사(ICN) — 핵심 사용자 ★
- **프로필**: 김선영, 41세, 요양병원 감염관리 전담간호사 (200병상, 경력 12년)
- **하루 업무**: 병동 순회·환경 점검, 감염 징후 모니터링, 환기·소독 지시, 적정성평가 증빙 작성, 보호자 응대
- **핵심 욕구(KBI)**: "집단감염 0건 유지 + 행정부담 최소화 + 적정성평가 점수 확보"
- **가장 큰 고통**: 위험을 미리 알 방법이 없고(사후 대응), 판단 근거가 직감이며, 수기 기록이 업무의 절반

### 페르소나 ② 병원장/운영이사 — 결재자 ★
- **프로필**: 이재호, 58세, 요양병원 이사장
- **핵심 욕구**: "집단감염 리스크 회피(휴원·소송·평판 손실) + 적정성평가 수가 가산"
- **구매 결정 기준**: ROI(리스크 보험 가치), 규제 준수, 신규 HW 비용 최소
- **가장 큰 고통**: 감염 1건이 수억 손실(휴원·배상·등급 하락)인데, 통제 수단이 없음

### 페르소나 ③ 보호자 — 신뢰 수혜자
- **프로필**: 박민지, 34세, 입소 부모를 둔 직장인
- **핵심 욕구**: "부모님이 안전한지 실시간으로 안심하고 싶다"
- **가장 큰 고통**: 면회 제한 상황에서 부모 상태를 알 길이 없는 불안 → Sentinel 보호자 PWA가 직접 대응

### Actor 정의서 (시스템 관점)
| Actor | 역할 | 시스템 상호작용 | 권한 |
|---|---|---|---|
| ICN(감염관리) | 위험 모니터·승인 | 대시보드 열람, 고위험 제어 승인 | 읽기+승인 |
| 시설관리(FM) | 가전·환기 운영 | 자동제어 모니터, 수동 오버라이드 | 읽기+제어 |
| 병원장 | 의사결정·결재 | KPI·증빙 리포트 열람 | 읽기 |
| 보호자 | 안심 수혜 | PWA 실시간 알림 수신 | 읽기(본인 병실) |
| 시스템(자동) | 오케스트레이션 | 위험등급→8종 가전 처방 | 자동 실행 |

## Page 5. 핵심 고객경험 — Pain Points · CAM

### Pain Points (ICN 기준, 우선순위순)
1. **사후 인지** — "감염 위험을 미리 알 방법이 전혀 없다. 증상 터지면 이미 늦다."
2. **근거 없는 판단** — "환기·소독 타이밍을 경험과 직감에 의존한다."
3. **행정부담** — "감염관리 활동을 일일이 수기로 기록하고 적정성평가용으로 정리한다."
4. **분절된 가전** — "공청기·에어컨·환기가 따로 놀아 통합 제어가 안 된다."
5. **보호자 불안 응대** — "전화로 일일이 '괜찮다'고 설명하는 데 시간을 쓴다."

### CAM (Customer Activity Map) — ICN 하루 동선 × Pain × Sentinel 개입
| 단계 | 활동 | 현재 Pain | Sentinel 개입 | 연결 화면 |
|---|---|---|---|---|
| 출근/인수인계 | 밤사이 상태 파악 | 데이터 분산 | 대시보드 전 병동 요약 | `/dashboard` 헤더 |
| 병동 순회 | 환기·온습도 점검 | 수기·직감 | tier 자동 표시·평면도 | 병동맵 모달 |
| 위험 인지 | 감염징후 판단 | 근거 부족 | **외부 유행 선제 경보 + 근거** | 외부신호 지도 |
| 대응 | 환기·소독·격리 | 수동 지시 | 위험등급 기반 가전 자동제어 | 가전 오케스트레이션 |
| 기록/보고 | 일지·적정성평가 | 행정부담 | **대응 이력 자동 로그·CSV** | 이벤트 로그·KPI |
| 보호자 응대 | 안심 설명 | 구두 반복 | PWA 실시간 안심 알림 | 보호자 앱 `/m` |

> 이 CAM은 **시연 동선 스크립트와 1:1 매핑** — 시연에서 각 단계를 화면으로 보여준다.

## Page 6. 핵심경험 컨셉 — 대표컨셉 정의서 · 시연모델 전략캔버스

### 대표 컨셉 정의서
> **컨셉 한 줄: "보이지 않는 감염 위험을, 보이게. 그리고 스스로 미리 대응하게."**
> (The Invisible, Made Visible — and Self-Responding)

**논리성 (인과 체인이 명확한가)**
```
외부 역학 선행신호(질병청·하수·OTC)
   → 실내 환경센서(CO2·PM·온습도) 융합
   → Wells-Riley/Rudnick-Milton 감염위험 5-Tier 산정
   → 위험등급 기반 8종 가전 차등 자동 처방 (하이브리드 거버넌스)
   → 대응 이력 적정성평가 증빙 자동 문서화
   → 보호자 실시간 안심
```
각 화살표가 코드로 실증됨(외부신호 boost → compute_tier → 가전제어 → rehva_results 적재 → PWA).

**창의성 (차별성)**
1. **외부 역학 융합 선제성** — 상용 제품 0건인 진짜 빈영역(벤치마킹 검증).
2. **벤더 무관 오케스트레이션** — 코웨이·삼성·LG를 단일 인터페이스로(한 줄 교체). 기존 가전 위에 SW만 얹어 신규 HW 0원.
3. **보호자 안심 PWA** — B2G/B2B 솔루션에 B2C 감성 가치 결합.

**충실성 (실제 구현됐는가)**
- 실센서→tier→코웨이 실제 제어 end-to-end 작동, 부하테스트 p95 60ms, 5종 시나리오 자동화, 9개 법령 컴플라이언스 매핑. (P8 개발과 직결)

### 시연모델 전략캔버스 (1장)
| 항목 | 내용 |
|---|---|
| **핵심 가치제안** | 유행 감지 시 공간을 선제 관리하고, 감염관리를 자동 증빙해 적정성평가 점수와 행정부담을 해결 |
| **타겟 고객** | 요양병원 ICN(사용자)·원장(결재자)·보호자(수혜자) |
| **핵심 활동** | 외부역학 선제경보 · 위험등급 가전 처방 · 컴플라이언스 자동화 |
| **핵심 자원** | UIS 역학 연계 · 실센서 · 벤더무관 가전 어댑터 · 9법령 매핑 |
| **수익 모델** | 구독(병상 규모별) + 초기 설정비 + PoC. 가치근거 = 적정성평가 수가 가산 + 집단감염 리스크 회피 |
| **차별화(해자)** | 외부역학 융합(무존재) · 벤더무관 · 파일럿 데이터 축적 · 규제안전 포지셔닝 |
| **시연 시나리오** | 정상 → 외부 유행 감지(센서 정상) → 선제 가동 → 위험 상승·승인 → 회복·자동 증빙 (75초) |
